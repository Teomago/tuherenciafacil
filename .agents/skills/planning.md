# Implementation Planning

## Overview
Write comprehensive implementation plans assuming the engineer has zero context. Each step must be a bite-sized task (2-5 minutes).

## Task Structure
Every task MUST follow this format:
- **Files:** Create/Modify/Test paths.
- **Steps:** Granular actions with code blocks.
- **TDD:** Include the failing test code in the first step.
- **Verification:** The exact command to run and expected output.
- **Commit:** The specific git command for that task.

## No Placeholders
- NEVER use "TBD", "TODO", or "implement later".
- Show the EXACT code to be written.
- If a method is modified, show the new version.

## Red Flags
- Steps like "Write tests for the above" (show the test).
- "Handle edge cases" (list the edge cases).
- "Similar to Task N" (repeat the code).
