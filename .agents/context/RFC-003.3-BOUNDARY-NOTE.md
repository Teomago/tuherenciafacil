# RFC-003.3 Boundary Note (for decision-file header)

Use this note verbatim or as the opening section of `.agents/decisions/RFC-003.3-decision.md`.

---

## Intent

`RFC-003.3` completes the **workflow-domain backend** required to operate succession cases safely and consistently before major UI funnel expansion.

## In scope

- Documents domain and R2-backed document lifecycle.
- Checklist generation and status updates tied to case progress.
- Notary-process tracking data structures and role restrictions.
- Phase-gate rules that block/allow workflow transitions.
- Payments domain foundation and integration boundary consistent with DEC-005.

## Out of scope

- App shell and component-system work (`RFC-004`).
- Onboarding/pre-payment screen orchestration (`RFC-005`).
- Broad UX polish unrelated to backend/admin verification.

## Non-negotiable execution guard

If a task requires behavior outside the above scope and is not explicitly in the decision file, stop and escalate. Do not absorb adjacent RFC scope.
