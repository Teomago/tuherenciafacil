# RFC-003.2 — Intake, Heirs & Assets

**Objective:** Implement the data-gathering layer for the succession process, covering the initial intake (pre-payment) and the granular data structures for heirs and assets (post-payment).

---

## 1. CaseIntake Collection (New)
**File:** `src/payload/collections/succession/CaseIntake/index.ts`
**Purpose:** A low-friction bridge to capture lead data before the "big" payment.

**Schema:**
- `member`: `relationship` -> `members`, required, read-only.
- `status`: `select` ['draft', 'submitted', 'paid', 'converted'], default: 'draft'.
- `tierElegido`: `select` ['estandar', 'premium', 'elite'], optional in schema (required for 'submitted').
- **Eligibility Filter Snapshot:**
  - `tieneTestamento`: `checkbox`, default: false (Read-only reference).
  - `acuerdoHerederos`: `select` ['si', 'no_sabe'], required.
- **Causante Data (High-level):**
  - `causanteNombre`: `text`, required.
  - `causanteCedula`: `text`, required.
  - `causanteFechaFallecimiento`: `date`, required.
  - `causanteCiudadFallecimiento`: `text`, required.
- **Estimated Scope:**
  - `herederosEstimados`: `number`, min: 1, required.
  - `valorEstimadoTotal`: `number`, COP, optional.
- `case`: `relationship` -> `cases`, optional (filled upon conversion).

---

## 2. Heirs Collection (New)
**File:** `src/payload/collections/succession/Heirs/index.ts`
**Purpose:** Granular data for every heir in a case.

**Schema:**
- `case`: `relationship` -> `cases`, required.
- `nombre`: `text`, required.
- `cedula`: `text`, required.
- `parentesco`: `select` (Hijo, Cónyuge, Padre, Hermano, Otro), required.
- `esFallecido`: `checkbox`, default: false.
- `esRepresentante`: `checkbox`, default: false (Used if the actual heir is deceased).
- `herederoOriginal`: `relationship` -> `heirs` (Required if `esRepresentante` is true).
- `representacionValidada`: `checkbox`, lawyer-only update.

---

## 3. Assets Collection (New)
**File:** `src/payload/collections/succession/Assets/index.ts`
**Purpose:** Granular data for the estate inventory.

**Schema:**
- `case`: `relationship` -> `cases`, required.
- `tipo`: `select` (Inmueble, Vehículo, Cuenta Bancaria, Acción/Sociedad, Otro), required.
- `valorEstimado`: `number`, required.
- `descripcion`: `textarea`, required.
- `identificador`: `text` (Placa, Matrícula, Número de cuenta), required.
- `investigacion`: `group`
  - `resultado`: `relationship` -> `media`.

---

## 4. API & Logic
- **Endpoint:** `POST /api/succession/convert-intake`
- **Logic:** Clones `CaseIntake` -> `Case`. Maps flat causante fields to `Case.causante` group. Sets `Case.status: active`. Sets `CaseIntake.status: converted`.
