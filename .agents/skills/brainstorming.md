# Brainstorming to Design

## The Hard Gate
**NO IMPLEMENTATION ACTIONS until a design is presented and approved.**

## Process
1.  **Explore Context:** Read files, docs, and recent commits.
2.  **Clarifying Questions:** One at a time. Purpose, constraints, success criteria.
3.  **Approaches:** Propose 2-3 approaches with trade-offs and a recommendation.
4.  **Design Presentation:** Present in logical sections (Architecture, Data, UI).
5.  **Design Doc:** Save to `docs/design/` and get final approval.
6.  **Transition:** Invoke `writing-plans` only after approval.

## Decomposition
If a project is too large for one spec, break it into sub-projects. Each sub-project follows the full cycle.
