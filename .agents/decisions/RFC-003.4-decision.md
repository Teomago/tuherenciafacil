## 1. Canonical Decisions

### 1.1 filterOptions pattern

All `filterOptions` in this RFC use `data.case` (top-level field) for case-scoping. The canonical pattern:

```typescript
filterOptions: ({ data }) => {
  if (!data?.case) return false
  const caseId = typeof data.case === 'object' ? data.case.id : data.case
  return { case: { equals: caseId } }
}
```

`filterOptions` protects the **admin UI picker only**. For security-critical fields (role assignments, heir representation), a matching `beforeChange` hook provides API-level enforcement.

### 1.2 Heir representation decision tree

When `esRepresentante === true` on a Heir:
1. `herederoOriginal` is **required** (server-side enforced).
2. `herederoOriginal` must belong to the **same case**.
3. `herederoOriginal` must have `esFallecido === true`.

Both `filterOptions` (admin UI) and `beforeChange` (API) enforce all three rules.

### 1.3 Intake eligibility

- `tieneTestamento === true` blocks **conversion only** (at the `convert-intake` endpoint), not intake saving.
- `herederosEstimados` is already protected by `min: 1, required: true` — no new code needed.
- `acuerdoHerederos` already excludes `'no'` from options — no new code needed.

### 1.4 Availability model

Paola (or future lawyers) creates availability slots freely. No hard-coded day/time rules.

- Slots define: date, start/end time, appointment duration, max appointments per window.
- Clients book into open slots. Booking is immediately confirmed (no approval flow).
- Appointments must be paid to be confirmed (ties into `payments`).
- Appointments work for authenticated members **and** guests without accounts.
- Clients cannot cancel but can reschedule with **24+ hours** notice before the meeting.
- Under 24 hours → rescheduling blocked.
- Paola can reschedule at any time (triggers email notification — email sending is out of scope for this RFC but the data model must support storing the reschedule event).

### 1.5 Existing implementations — verify, don't reimplement

The following already exist from RFC-003.3 and must be verified, not rewritten:

- Poder gate (Phase 3→4) in `validatePhaseTransition.ts`
- `herederosEstimados` schema validation in `CaseIntake/index.ts`
- `visibleToClient` field in `Payments/index.ts`
- Heir checklist trigger (`esRepresentante`) in `generateChecklistItems.ts`

If verification reveals a gap, document it and fix it. If no gap, mark it as confirmed.

```typescript
import type { CollectionBeforeChangeHook } from 'payload'
import { APIError } from 'payload'

export const validateRoleAssignments: CollectionBeforeChangeHook = async ({
  data,
  req,
  operation,
  originalDoc,
}) => {
  if (data.responsable && data.responsable !== originalDoc?.responsable) {
    const member = await req.payload.findByID({
      collection: 'members',
      id: typeof data.responsable === 'object' ? data.responsable.id : data.responsable,
      req,
    })
    if (member.role !== 'cliente') {
      throw new APIError('El responsable debe ser un miembro con rol cliente.', 422)
    }
  }

  if (data.abogadoAsignado && data.abogadoAsignado !== originalDoc?.abogadoAsignado) {
    const member = await req.payload.findByID({
      collection: 'members',
      id: typeof data.abogadoAsignado === 'object' ? data.abogadoAsignado.id : data.abogadoAsignado,
      req,
    })
    if (member.role !== 'abogado' || member.isActive === false) {
      throw new APIError('El abogado asignado debe ser un abogado activo.', 422)
    }
  }

  return data
}
```

```typescript
import type { CollectionBeforeChangeHook } from 'payload'
import { APIError } from 'payload'

export const validateHeirRepresentation: CollectionBeforeChangeHook = async ({
  data,
  req,
}) => {
  if (!data.esRepresentante) return data

  if (!data.herederoOriginal) {
    throw new APIError(
      'Un heredero representante debe tener vinculado al heredero original fallecido.',
      422,
    )
  }

  const originalId =
    typeof data.herederoOriginal === 'object'
      ? data.herederoOriginal.id
      : data.herederoOriginal

  const original = await req.payload.findByID({
    collection: 'heirs',
    id: originalId,
    req,
  })

  if (!original.esFallecido) {
    throw new APIError('El heredero original debe estar marcado como fallecido.', 422)
  }

  const originalCaseId =
    typeof original.case === 'object' ? original.case?.id : original.case
  const thisCaseId =
    typeof data.case === 'object' ? data.case?.id : data.case

  if (originalCaseId !== thisCaseId) {
    throw new APIError('El heredero original debe pertenecer al mismo caso.', 422)
  }

  return data
}
```

```typescript
import type { CollectionBeforeChangeHook } from 'payload'
import { APIError } from 'payload'
import { isAdminUser, isMemberUser } from '@/lib/auth/typeGuards'

export const validateReschedule: CollectionBeforeChangeHook = async ({
  data,
  req,
  operation,
  originalDoc,
}) => {
  if (operation !== 'update') return data
  if (!originalDoc?.fechaAgendada) return data

  // Check if fechaAgendada is changing (rescheduling)
  if (data.fechaAgendada && data.fechaAgendada !== originalDoc.fechaAgendada) {
    const user = req.user

    // Lawyers and admins can reschedule at any time
    if (isAdminUser(user)) return data
    if (isMemberUser(user) && user.role === 'abogado' && user.isActive !== false) {
      data.rescheduledAt = new Date().toISOString()
      data.rescheduledBy = 'lawyer'
      return data
    }

    // Clients: enforce 24-hour minimum
    const appointmentDate = new Date(originalDoc.fechaAgendada)
    const now = new Date()
    const hoursUntilAppointment =
      (appointmentDate.getTime() - now.getTime()) / (1000 * 60 * 60)

    if (hoursUntilAppointment < 24) {
      throw new APIError(
        'No es posible reagendar con menos de 24 horas de anticipación.',
        422,
      )
    }

    data.rescheduledAt = new Date().toISOString()
    data.rescheduledBy = 'client'
  }

  return data
}
```
