# Audit — RFC-003.2: Intake, Heirs & Assets

**Auditor:** Claude Code (Reconstructed)  
**Date:** 2026-04-11  
**Verdict:** REQUIRES CHANGES IN DECISION

## Findings

### CRIT-1: Missing Lawyer-Only Access
Fields like `representacionValidada` in Heirs and `notasInternas` must be strictly restricted to `role: abogado` or `admin`. Clients must not be able to validate their own representation status.

### CRIT-2: Representation Logic Ambiguity
If an heir is deceased, the representative must link back to them. The spec didn't specify that the `heirs` collection needs to allow self-reference or a clear hierarchy.

### MAJ-1: Field Naming Inconsistency
The spec uses `caseCreated`. The canonical `COLLECTIONS.md` uses `case`. We must rename to `case` to avoid schema drift.

### MAJ-2: Tier Selection UX
`tierElegido` is marked as `required`. This prevents users from saving a lead "draft" before they are ready to pick a price.
**Resolution:** Make it optional at the DB level, but enforce it via a `beforeChange` hook only when the status is `submitted`.

### MIN-1: Missing Eligibility Persistence
The `tieneTestamento` and `acuerdoHerederos` flags should be copied to the `Case` object during conversion for permanent legal record.
