# RFC-003.1 — Core Identity & Maintenance Update

**Objective:** Synchronize the tech stack with the latest stable versions and implement the fundamental identity layer (Members, Cases, Appointments) to support the succession workflow and credit system.

---

## 1. Maintenance Update (Pre-requisite)
**Purpose:** Ensure the platform uses the most stable APIs for Payload 3.x and Next.js 16.

**Changes in `package.json`:**
- `next`: `16.2.1` → `16.2.3`
- `payload`: `3.80.0` → `3.82.0` (and all `@payloadcms/*` plugins)
- `react`: `19.2.4` → `19.2.5`
- `react-dom`: `19.2.4` → `19.2.5`
- `typescript`: `5.7.3` → `6.0.2`
- `drizzle-orm`: `0.45.1` → `0.45.2`
- `next-intl`: `4.8.3` → `4.9.0`

---

## 2. Members Collection (Update)
**File:** `src/payload/collections/settings/Members/index.ts`

**Schema Changes:**
- **Add `role`**: `select` ['cliente', 'abogado'], required, default: 'cliente'.
- **Add `isActive`**: `checkbox`, default: true. (Admin-only editable).
- **Add `cedula`**: `text`, required, unique. Regex validation: `^[0-9.]+$`, minLength 6, maxLength 12.
- **Add `telefono`**: `text`, required. Regex: `^3[0-9]{9}$` (Colombian mobile format).
- **Add `ciudad`**: `text`, required.
- **Add `creditoAcumulado`**: `number`, default: 0, min: 0. (Editable by Admin/Abogado).
- **Add `isVerified`**: `checkbox`, default: false.

**Access Control:**
- `read`: Own profile always. Abogados can read Clientes assigned to their cases.
- `update`: Own profile (except `role`, `isActive`, `isVerified`, `creditoAcumulado`). Admin can update all.

---

## 3. Cases Collection (New)
**File:** `src/payload/collections/succession/Cases/index.ts`

**Schema:**
- `caseNumber`: `text`, required, unique, read-only.
- `status`: `select` ['active', 'completed'], default: 'active'.
- `currentPhase`: `number`, min: 0, max: 8, default: 0.
- `tier`: `select` ['estandar', 'premium', 'elite'], required, default: 'estandar'.
- `responsable`: `relationship` -> `members`, required.
- `abogadoAsignado`: `relationship` -> `members`, optional.
- `causante`: `group` (nombre, cedula, fechaFallecimiento, ciudadFallecimiento).
- `notasInternas`: `richText` (Lawyer only).
- `notasAlCliente`: `richText` (Client visible).

**Hooks:**
- `beforeChange`: Generate `caseNumber` using format `SUC-YYYYMMDD-HHMM`.

---

## 4. Appointments Collection (New)
**File:** `src/payload/collections/succession/Appointments/index.ts`

**Schema:**
- `member`: `relationship` -> `members`, required.
- `tipo`: `select` ['consulta_virtual', 'consulta_presencial', 'mediacion_virtual', 'mediacion_presencial', 'caso_virtual', 'caso_presencial'], required.
- `monto`: `number`, required.
- `status`: `select` ['pendiente_pago', 'pagada', 'realizada', 'cancelada'], default: 'pendiente_pago'.
- `autorizarCredito`: `checkbox`, default: false.
- `creditoApplied`: `checkbox`, default: false.

**Hooks:**
- `afterChange`: If `autorizarCredito` changes to `true` AND `status` is `realizada`, add `monto` to `Member.creditoAcumulado`.
- `afterChange`: If `autorizarCredito` changes to `false` AND `creditoApplied` is `false`, subtract `monto` from `Member.creditoAcumulado`.

---

## 5. Translations
New keys for `es.json` and `en.json` in the `App` namespace:
- `role`, `cedula`, `telefono`, `ciudad`, `creditoAcumulado`, `isActive`.
- `caseNumber`, `currentPhase`, `tier`, `responsable`, `abogadoAsignado`.
- `appointmentType`, `autorizarCredito`, `monto`.

---

## 6. Success Criteria
1. `pnpm build` passes with the updated stack.
2. `Members` collection reflects new fields in `/admin`.
3. `Cases` and `Appointments` collections are registered and visible in `/admin`.
4. **TDD:** `Appointments` hook correctly updates `Member.creditoAcumulado` (verified with Vitest).
5. **TDD:** `caseNumber` is automatically generated on `Case` creation (verified with Vitest).
6. **Access:** `isActive: false` lawyers cannot access the "Pool" or any cases (verified with Playwright).

---

## Out of Scope
- Document uploads (RFC-003.3).
- Real Wompi payments (RFC-008).
- UI screens beyond the `/admin` panel.
