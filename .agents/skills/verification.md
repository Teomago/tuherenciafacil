# Verification Before Completion

## The Iron Law
**NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE.**

## The Process
1.  **Identify:** What command proves this claim?
2.  **Run:** Execute the full command (fresh, complete).
3.  **Read:** Check exit code, failure count, and logs.
4.  **Report:** State the claim WITH the evidence.

## Red Flags
- Using "should", "probably", "seems to".
- Expressing satisfaction ("Done!", "Perfect!") before running verification.
- Trusting agent success reports without checking the `git diff`.
